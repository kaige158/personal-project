# 项目说明（给 Claude 的持久指令）

## 项目作者背景

- **身份**：广西农业职业技术大学，计算机应用工程专业，**2023 级**本科在读，当前为**大三**学生。
- **课程依据**：代码与文档应贴合本校 **2023 级入学培养方案** 中已学、正在学的课程内容，不引入明显超出该阶段教学范围的技术与概念。
- **项目目的**：个人作业 / 课程设计 / 毕业相关实践，需要我能**读懂每一部分逻辑**，并在答辩时**用自己的话讲清楚**设计与实现。

## 代码与实现原则（必须遵守）

### 1. 知识维度对齐

优先使用培养方案中常见、我能在课堂上讲清的技术栈，例如（按课程实际选用，勿堆砌）：

- 程序设计：C / Java 基础语法，面向对象入门
- Web：HTML、CSS、JavaScript；后端可用 Servlet、SSM（Spring + SpringMVC + MyBatis）或课程指定的框架
- 数据库：MySQL，基础 SQL（增删改查、简单联表）
- 其他可能课程：计算机网络基础、软件工程基础、Android 入门、小程序基础等

**避免**（除非我明确要求）：微服务、Docker/K8s、复杂设计模式全家桶、冷门框架、过度抽象的架构、我看不懂的“企业级最佳实践”堆砌。

### 2. 可读性优先

- 结构清晰：按“界面 → 控制/接口 → 业务 → 数据访问”分层，文件夹与类名见名知意。
- **关键逻辑必须有中文注释**，说明“这一步在干什么、为什么这样写”。
- 单文件、单函数不宜过长；复杂流程用简短步骤注释标出 ①②③。
- 变量、方法命名用常见英文单词，避免晦涩缩写。

### 3. 便于答辩

- 实现方案应能在答辩中用 3～5 分钟讲清：**需求 → 总体结构 → 核心功能怎么实现 → 数据库表设计思路**。
- 不写“为了炫技”的代码；每一处技术选型应能回答：**这门课里学过 / 实验里用过**。
- 若必须使用稍难的技术，须在注释或说明中写一句：**为什么用、对应哪门课或哪个知识点**。

### 4. 与 AI 协作时的默认行为

- 用**简体中文**解释思路、修改说明和答辩要点。
- 改代码前先说明**要改什么、为什么**；改完后用简短条目总结**我答辩时该怎么讲**。
- 不擅自引入大型依赖或重构整个项目；小步修改，保持我能跟得上。
- 生成 README、实验报告提纲、答辩 PPT 要点时，语言风格符合**本科课程作业**，朴实、具体，少空话。

## 语言

- 与用户交流、注释、文档说明：**默认简体中文**。
- 代码中的类名、API、路径等技术标识保持常规英文写法。
